# UptimeRobot "Coming soon" 상태 해결 가이드

## 현재 상황

- ✅ 모니터 생성 완료
- ⏳ "Last check: Coming soon" 표시
- ⏳ 첫 체크 대기 중

## 의미

"Coming soon"은:
- 모니터가 방금 생성되었거나
- 아직 첫 체크가 실행되지 않았음을 의미합니다
- **정상적인 상태**이며, 첫 체크까지 최대 5분이 걸릴 수 있습니다

## 대기 시간

- **예상 시간**: 1-5분
- **체크 간격**: 5분마다
- **첫 체크**: 모니터 생성 후 최대 5분 내

## 대기 중 확인 사항

### 1단계: 서버 직접 테스트 (지금 바로 확인 가능)

브라우저에서 다음 URL을 직접 접속하여 서버가 정상 작동하는지 확인:

```
https://bangguy.onrender.com/api/ip-status
```

**예상 응답:**
```json
{"ip":"...","office":true}
```

**정상 응답이면:**
- ✅ 서버는 정상 작동 중
- ✅ UptimeRobot이 곧 체크를 시작할 것입니다
- ⏳ 몇 분만 기다리면 됩니다

**오류 응답이면:**
- ❌ 서버 문제 가능
- ❌ Render 로그 확인 필요

### 2단계: 다른 엔드포인트 테스트

**루트 페이지:**
```
https://bangguy.onrender.com/
```

**관리자 페이지:**
```
https://bangguy.onrender.com/admin.html
```

### 3단계: 모니터 설정 재확인

UptimeRobot 대시보드에서 모니터 설정 확인:

1. 모니터 클릭 → "Edit" 클릭
2. 다음 항목 확인:
   - **Type**: `HTTP(s)` ✅
   - **URL**: `https://bangguy.onrender.com/api/ip-status` ✅
     - 끝에 슬래시(`/`) 없어야 함
     - `https://` 사용
   - **Interval**: `5 minutes` ✅
3. "Save Changes" 클릭 (변경사항이 있었다면)

## 예상 타임라인

```
현재: 모니터 생성 완료, "Coming soon" 상태
↓
1-5분 후: 첫 체크 실행
↓
체크 성공: "Up" (초록색) 상태로 변경
↓
5분마다: 자동 체크 반복
```

## 확인 방법

### 방법 1: UptimeRobot 대시보드 새로고침

1. UptimeRobot 대시보드 새로고침 (F5)
2. 모니터 상태 확인
3. "Last check" 시간 업데이트 확인

### 방법 2: 몇 분 후 다시 확인

1. 5분 후 다시 확인
2. "Last check" 시간이 업데이트되었는지 확인
3. 상태가 "Up" (초록색)으로 변경되었는지 확인

## 문제 해결

### 시나리오 1: 5분 후에도 "Coming soon"

**원인:**
- 모니터 설정 오류
- 서버 응답 오류
- UptimeRobot 서비스 문제

**조치:**
1. 브라우저에서 `/api/ip-status` 직접 접속 테스트
2. 서버 로그 확인 (Render 대시보드)
3. 모니터 설정 재확인
4. 모니터 삭제 후 재생성

### 시나리오 2: 첫 체크 후 "Down" 상태

**원인:**
- 서버 오류
- URL 오류
- 네트워크 문제

**조치:**
1. 브라우저에서 `/api/ip-status` 직접 접속 테스트
2. 서버 로그 확인
3. 모니터 URL 정확히 확인

### 시나리오 3: 첫 체크 후 "Up" 상태

**상태:**
- ✅ 정상 작동 중
- ✅ 슬립 모드 방지 작동
- ✅ 추가 조치 불필요
